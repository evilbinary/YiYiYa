# 架构设计

## 系统架构

YiYiYa 采用微内核架构设计，具有以下特点：

### 微内核架构

```
+------------------+
|   用户空间进程    |
+------------------+
         |
         | IPC (进程间通信)
         |
+------------------+
|    微内核核心     |
|  - 调度器        |
|  - 内存管理      |
|  - IPC          |
|  - 中断处理      |
+------------------+
         |
         | 设备接口
         |
+------------------+
|    设备驱动       |
|  - 网络驱动      |
|  - 存储驱动      |
|  - 显示驱动      |
+------------------+
```

### 核心组件

#### 1. 内核层 (Kernel)
- **调度器**: 优先级调度、时间片轮转
- **内存管理**: 页式内存管理、虚拟内存
- **进程管理**: 进程创建、切换、通信
- **中断处理**: 硬件中断、软件中断

#### 2. 驱动层 (Drivers)
- **网络驱动**: BCM2837、E1000 等
- **存储驱动**: SD卡、硬盘等
- **显示驱动**: LCD、VGA 等
- **输入驱动**: 键盘、鼠标、触摸屏

#### 3. 系统服务层 (Services)
- **文件系统**: FAT32、ext2 等
- **网络服务**: TCP/IP 协议栈
- **图形服务**: GUI 框架

#### 4. 应用层 (Applications)
- Shell
- GUI 应用
- 开发工具

### 模块化设计

每个功能模块独立编译，可动态加载：

```
duck/modules/
├── net/          # 网络模块
├── fat/          # 文件系统
├── lcd/          # 显示驱动
├── serial/       # 串口驱动
└── ...
```

### 多平台支持

支持多种硬件平台：

| 平台 | 架构 | 状态 |
|------|------|------|
| Raspberry Pi 3 | ARM64 | ✅ 支持 |
| Raspberry Pi 2 | ARMv7 | ✅ 支持 |
| STM32F4xx | ARMv7-M | ✅ 支持 |
| V3S | ARMv7 | ✅ 支持 |
| QEMU | x86/ARM | ✅ 支持 |

### 进程间通信 (IPC)

支持多种 IPC 机制：
- 消息队列
- 共享内存
- 信号量
- 管道

### 内存管理

#### 虚拟内存
- 页式管理 (4KB 页面)
- 虚拟地址空间隔离
- 写时复制 (COW)

#### 物理内存
- 内存池管理
- DMA 内存区域
- 内存映射 I/O

### 调度策略

1. **优先级调度**: 实时任务优先
2. **时间片轮转**: 公平分配 CPU 时间
3. **多核调度**: SMP 支持

### 设备模型

统一的设备模型：

```c
typedef struct device {
    char* name;
    u32 id;
    u32 type;
    size_t (*read)(device_t*, void*, size_t);
    size_t (*write)(device_t*, const void*, size_t);
    int (*ioctl)(device_t*, u32, void*);
    void* data;
} device_t;
```

### 文件系统

支持多种文件系统：
- FAT32 (通过 FatFs)
- ext2 (开发中)
- 虚拟文件系统 (VFS)

### 安全性

- 用户态/内核态隔离
- 地址空间隔离
- 权限控制
- 能力机制 (Capability)

## 设计原则

1. **微内核**: 保持内核最小化，服务用户态运行
2. **模块化**: 功能模块独立，可配置
3. **可移植**: 支持多种硬件平台
4. **实时性**: 支持实时任务调度
5. **安全性**: 进程隔离、权限控制
